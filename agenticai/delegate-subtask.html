# antigravity — Delegate to the Antigravity CLI

The `agy` CLI runs Antigravity (an agentic assistant) non-interactively. Use it to hand
off a bounded subtask, capture the result from stdout, and continue the main task in
this session.

## Use when

- A subtask is well-scoped and expressible as a single prompt — translation,
  summarization, drafting, extraction, classification, or one-off code/text generation.
- You want to offload work that is cheaper or faster handled outside this session, then
  consume the result here.
- You want a second model's take on a self-contained question (Antigravity exposes
  Gemini, Claude, and GPT-OSS models).

## Do not use when

- The work depends on this session's accumulated context, open files, or tools — keep
  that here.
- You need tight interactive iteration, or to drive the tool calls yourself.
- It's trivial enough that doing it inline is faster than shelling out.

## How to call

Print mode (`-p`, alias for `--print` / `--prompt`) runs one prompt and prints the
response to stdout:

```bash
agy -p "<your prompt here>"
```

Pipe larger input on stdin and reference it in the instruction:

```bash
cat <file> | agy -p "<instruction describing what to do with the input>"
```

Or give Antigravity read access to files instead of pasting them:

```bash
agy --add-dir . -p "Review app/conf/.../FooController.scala and list any unhandled error cases."
```

### Useful options

- `--model "<name>"` — pick the model; list options with `agy models` (e.g.
  *Gemini 3.5 Flash* for cheap/fast, *Claude Opus 4.6 (Thinking)* for hard reasoning).
- `--add-dir <dir>` — add a directory to Antigravity's workspace (repeatable) so it can
  read files itself.
- `--print-timeout <dur>` — raise the wait for long tasks (default 5m).
- `-c` / `--continue`, `--conversation <id>` — continue a prior delegation when one
  prompt isn't enough.

### Examples

```bash
# Translate to English, output only the translation
agy -p "Translate the following into English, output only the translation:\n<source text>"

# Summarize a file
cat notes.md | agy -p "Summarize the following in 5 bullet points."

# Cheap one-off generation on a fast model
agy --model "Gemini 3.5 Flash (Low)" -p "Write a regex for an RFC 5322 email. Output only the regex."
```

## Rules

1. **Scope the prompt fully.** `agy -p` is non-interactive — include everything it needs
   in one prompt (or use `-c` to continue an existing delegation).
2. **Delegate, then continue here.** Antigravity handles the offloaded subtask; the
   surrounding work, decisions, and edits stay in this session.
3. **Constrain the output.** Ask for exactly the format you want ("output only …") so the
   result is easy to consume programmatically.
4. **Match the model to the job.** A fast/cheap model for simple offloads, a stronger
   thinking model for hard reasoning.
5. **Surface failures.** If `agy` errors or is unavailable, report that plainly rather
   than guessing at the result.

# antigravity — Delegate to the Antigravity CLI

The `agy` CLI runs Antigravity (an agentic assistant) non-interactively. Use it to hand
off a bounded subtask, capture the result from stdout, and continue the main task in
this session.

## Use when

- A subtask is well-scoped and expressible as a single prompt — translation,
  summarization, drafting, extraction, classification, or one-off code/text generation.
- You want to offload work that is cheaper or faster handled outside this session, then
  consume the result here.
- You want a second model's take on a self-contained question (Antigravity exposes
  Gemini, Claude, and GPT-OSS models).

## Do not use when

- The work depends on this session's accumulated context, open files, or tools — keep
  that here.
- You need tight interactive iteration, or to drive the tool calls yourself.
- It's trivial enough that doing it inline is faster than shelling out.

## How to call

Print mode (`-p`, alias for `--print` / `--prompt`) runs one prompt and prints the
response to stdout:

```bash
agy -p "<your prompt here>"
```

Pipe larger input on stdin and reference it in the instruction:

```bash

```

Or give Antigravity read access to files instead of pasting them:

- `--print-timeout <dur>` — raise the wait for long tasks (default 5m).
